# Evaluator Card Template

Use this skeleton for every major empirical surface in the paper. Each section
heading must appear verbatim in generated cards so CI or manual review can grep
for completeness.

```markdown
# Evaluator Card: <surface name>

## Purpose

## Applicability domain

## Native benchmark assumptions

## Replay environment

## Strengthened validators

## Policy assumptions

## Semantic obligations

## Denominators and invalid handling

## Known false-positive risks

## Known false-negative risks

## Reproduction command

## Evidence bundle paths
```
